# Документация проекта Control Habits

Документация требований и системного дизайна сервиса для трекинга дел и привычек в течение дня (веб + Telegram-бот).

## Структура документов

| Документ | Описание |
|----------|----------|
| [PRD](prd.md) | Цели продукта, персоны, сценарии, границы MVP, метрики успеха |
| [Требования](requirements.md) | Функциональные и нефункциональные требования, граничные случаи |
| [Доменная модель](domain-model.md) | Сущности, состояния событий, инварианты |
| [Системный дизайн](system-design.md) | Архитектура, сервисы, потоки данных, хранилище, планировщик |
| [Компоненты](modules.md) | Разбиение на модули с интерфейсами для поэтапной реализации |
| [UI: Telegram](ui-telegram.md) | Схемы экранов и клавиатур бота |
| [UI: Веб](ui-web.md) | Схемы страниц веб-приложения |
| [Промпты по подзадачам](prompts.md) | Готовые формулировки для агентов и чата по каждой подзадаче |
| [CI/CD](ci-deploy/ci-cd.md) | Branch protection, ручной релиз |
| [Деплой](ci-deploy/deploy.md) | Docker Compose на прод, секреты, миграции Alembic |

## Расположение

Вся документация хранится в папке `docs/` в корне репозитория.
