# Product Requirements Document (PRD)

## 1. Цели продукта

**Продукт**: сервис для трекинга дел и привычек в течение дня.

**Цель**: дать пользователю возможность задать распорядок дня и получать в Telegram напоминания с фиксацией факта выполнения (и времени ответа), а также быстро отмечать интервальные активности (например, «смотрю YouTube») кнопками в боте.

**Ключевая ценность**:
- Напоминания по расписанию без необходимости помнить самому.
- Фиксация **фактического времени** ответа (когда нажал), а не только планового.
- Быстрый трекинг «сейчас делаю X» без захода на сайт.

## 2. Персоны

**Основная персона — «организованный пользователь дня»**
- Хочет следовать распорядку (подъём, завтрак, учёба, тренировка и т.д.).
- Готов один раз настроить расписание на сайте и потом взаимодействовать в основном через бота.
- Использует Telegram ежедневно; бот для него — естественный канал напоминаний.
- Может забыть нажать «закончил»; нужен список незавершённых активностей, чтобы быстро закрыть сессию.

**Вторичная персона**
- Пользователь, который хочет только считать время по активностям (hotkey-кнопки) без расписания. MVP это поддерживает частично: расписание можно не заполнять, использовать только кнопки и отчёт за день.

## 3. Сценарии использования (MVP)

### 3.1 Онбординг
1. Пользователь заходит на сайт.
2. Видит предложение «Войти через Telegram».
3. Нажимает, получает код и ссылку на бота (`t.me/<bot>?start=<code>`).
4. Открывает бота, нажимает Start (или вводит `/start <code>`).
5. Бот подтверждает привязку; сайт обновляется и открывает настройки.

### 3.2 Настройка расписания (веб)
1. Пользователь в редакторе расписания добавляет элементы: **дела** (одна точка времени, например «07:00 Подъём») и **события** (блок времени, например «09:00–11:00 Учёба»).
2. Указывает дни недели для каждого элемента.
3. Задаёт часовой пояс и при необходимости тихие часы/вкл пушей.

### 3.3 Настройка быстрых кнопок (веб)
1. Пользователь создаёт активности (например, «YouTube», «Работа»).
2. Добавляет нужные в список hotkey-кнопок с подписью и порядком.
3. Эти кнопки появятся в боте.

### 3.4 День в боте — пуши по расписанию
1. В плановое время бот присылает сообщение.
2. **Дело**: «Сделал [название]?» — кнопки «Сделал» / «Не сделал» / «Пропустить».
3. **Событие (старт)**: «Начал [название]?» — «Начал» / «Не начал».
4. **Событие (финиш)**: «Закончил [название]?» — «Закончил» / «Пропустил».
5. При нажатии система сохраняет `planned_at` и `responded_at` (время нажатия).

### 3.5 День в боте — быстрые кнопки
1. Пользователь нажимает кнопку активности (например, «YouTube») — стартует сессия (`started_at`).
2. Повторное нажатие или кнопка «Закончить» завершает сессию (`ended_at`).
3. Команда `/active` или кнопка «Что сейчас идёт» показывает список незавершённых активностей с кнопками «Закончить» по каждой.

### 3.6 Просмотр результатов (веб)
1. Пользователь открывает отчёт за день: что было запланировано, какие ответы даны (и когда), длительности интервалов по hotkey-активностям.

## 4. Границы MVP / не-MVP

**В MVP**:
- Один шаблон расписания на пользователя (с повторением по дням недели).
- Авторизация только через привязку «сайт + бот» по одноразовому коду.
- Пуши: дело (один вопрос), событие (два вопроса — старт и финиш).
- Hotkey-кнопки: старт/стоп сессии, список активных сессий и завершение из списка.
- Минимальный отчёт за день на вебе.

**Не в MVP (расширения)**:
- Несколько шаблонов (будни/выходные), исключения по датам.
- Snooze, grace period, автозавершение события по плановому концу.
- Telegram Login Widget на сайте (альтернатива коду в боте).
- Аналитика: цели, стрики, графики.
- Импорт/экспорт расписания (YAML/JSON).

## 5. Метрики успеха

- **Принятие**: доля пользователей, завершивших онбординг (привязали бота и зашли на сайт после этого).
- **Использование**: доля дней с хотя бы одним ответом на пуш или хотя бы одной hotkey-сессией.
- **Качество данных**: доля пушей с зафиксированным ответом (без «пропустить» по умолчанию) — как косвенная метрика вовлечённости.
- **Надёжность**: отсутствие дублей записей при повторных нажатиях (идемпотентность), успешная доставка пушей в заданное время (в пределах допустимой задержки).
